## 9. STRIKER SELECTION

### 9.1 Purpose
Select opening batsman (Striker) for first delivery.

### 9.2 Striker Selection Flow
```
Batting Team Captain View → See Playing XI → Select Opening Batsman → 
Confirm Striker → Ready for Non-Striker Selection
```

### 9.3 UI Components

#### 9.3.1 Player Selection Interface
```
┌──────────────────────────────────┐
│  SELECT OPENING STRIKER          │
│  Match: TMA vs TMB               │
│  Batting Team: Team A             │
│                                   │
│  Playing XI:                      │
│  ☐ Player 1 (Batsman)            │
│  ☑ Player 2 (Batsman) ✓ SELECTED │
│  ☐ Player 3 (All-rounder)        │
│  ...                              │
│                                   │
│  [CONFIRM STRIKER]               │
└──────────────────────────────────┘
```

#### 9.3.2 Striker Player Card
- Player Name & Jersey Number
- Batting Hand (Right/Left)
- Batting Position
- Recent Stats:
  - Runs in similar format
  - Strike rate
  - Form (good/average/poor)
- Player Avatar/Photo

#### 9.3.3 Position Indicator
- **Striker Position**: Receiving ball (highlighted)
- **Non-Striker Position**: Other end
- Visual field diagram

### 9.4 Selection Constraints
- Must be from batting team's Playing XI
- Must be confirmed player
- Typically position 1 (opener)
- Typically batsman/all-rounder

### 9.5 Data Model
```javascript
{
  matchId: ObjectId,
  
  striker: {
    playerId: ObjectId,
    playerName: String,
    jerseyNumber: Number,
    battingHand: 'right' | 'left',
    
    selectedAt: timestamp,
    selectedBy: ObjectId (captain),
    status: 'selected' | 'batting' | 'out'
  },
  
  strikerConfirmedAt: timestamp
}
```

### 9.6 Real-time Events
- Socket event: `striker:selection_started`
- Socket event: `striker:selected`
- Socket event: `striker:confirmed`

### 9.7 API Endpoints
```
GET /api/matches/{id}/playing-xi/batting
POST /api/matches/{id}/striker/select
PUT /api/matches/{id}/striker/confirm
```

## 11. TOURNAMENT SYSTEM

### 11.1 Purpose
Manage multi-match tournaments with points table, fixtures, and winners.

### 11.2 Tournament Types
- **Round-Robin**: Each team plays every other team
- **Knockout**: Elimination format (winners advance)
- **Group + Knockout**: Group stage followed by knockouts
- **League**: Points-based system over multiple rounds

### 11.3 Tournament Components

#### 11.3.1 Tournament Creation

- Start and end dates
- Venue/venues
- Number of teams
- Tournament format
- Match schedule
- Prize pool
- Registration fee

#### 11.3.2 Team Registration

- Player list (with phone numbers)
- Jersey colors
- Team logo/badge
- Registration confirmation

```
### 19.2 Socket.IO Events Summary

**Match Events**
- `match:join` → Player joins match room
- `ball:scored` → Ball has been scored
- `wicket:taken` → Wicket has fallen
- `over:completed` → Over completed
- `innings:completed` → Innings ended
- `match:completed` → Match finished

**Commentary Events**
- `commentary:add` → Add commentary
- `commentary:added` → Commentary added (broadcast)

**Notifications**
- `notification:new` → New notification received
- `notification:read` → Mark notification as read
```
